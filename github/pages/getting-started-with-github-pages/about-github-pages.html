About GitHub Pages

GitHub Pages 是一种静态站点托管服务，可直接从 GitHub 上的存储库（repository）获取 HTML、CSS 和 JavaScript 文件，可选择通过构建过程运行文件，并发布网站。 您可以在 GitHub Pages examples collection(此链接网页未翻译) 中查看 GitHub Pages 站点的示例。

您可以在 GitHub 的 github.io 域或您自己的自定义域上托管您的站点。 更多信息请参阅"Using a custom domain with GitHub Pages."(此链接网页未翻译)。

如果您的项目站点是从使用 GitHub Enterprise Cloud 的组织拥有的私有或内部仓库发布的，您可以管理站点的访问控制。更多信息请参阅 "Changing the visibility of your GitHub Pages site"(此链接网页未翻译)。

To get started, see "Creating a GitHub Pages site"(此链接网页未翻译)。

组织所有者可以禁止从组织的仓库中发布 GitHub Pages 站点。 更多信息请参阅 "Managing the publication of GitHub Pages sites for your organization"(此链接网页未翻译)。

 

 

Types of GitHub Pages sites

GitHub Pages 站点分为三种类型：项目、用户和组织。 项目站点连接到托管在 GitHub 上的特定项目，例如 JavaScript 库或配方集合。 用户和组织站点连接到特定的 GitHub 帐户。

要发布用户站点，您必须创建一个由您的用户帐户拥有的名为 <username>.github.io 的存储库。 要发布组织站点，您必须创建一个由名为 <organization>.github.io 的组织拥有的存储库。 除非您使用自定义域，否则用户和组织站点可从 http(s)://<username>.github.io 或 http(s)://<organization>.github.io 获得。

项目站点的源文件与其项目存储在同一存储库中。除非您使用自定义域，否则项目站点位于 http(s)://<username>.github.io/<repository> 或 http(s)://<organization>.github.io/<repository>.

如果您私下发布您的网站，您网站的 URL 将有所不同。 更多信息请参阅 "Changing the visibility of your GitHub Pages site"(此链接网页未翻译)。

有关自定义域如何影响站点 URL 的更多信息，请参阅 "About custom domains and GitHub Pages"(此链接网页未翻译)。

您只能为 GitHub 上的每个帐户创建一个用户或组织站点。 项目站点，无论是由组织还是用户帐户拥有，都是无限制的。

 

 

Publishing sources for GitHub Pages sites

GitHub Pages 站点的发布源是存储站点源文件的分支和文件夹。

警告：默认情况下，GitHub Pages 站点在 Internet 上是公开可用的，即使站点的存储库是私有的或内部的。如果您的项目站点是从使用 GitHub Enterprise Cloud 的组织拥有的私有或内部仓库发布的，您可以管理站点的访问控制。否则，如果您站点的存储库中有敏感数据，您可能需要在发布前删除这些数据。更多信息请参阅"About repository visibility"(此链接网页未翻译) 和 "Changing the visibility of your GitHub Pages site"(此链接网页未翻译)。

如果您的仓库中存在默认发布源，GitHub Pages 将自动从该源发布站点。用户和组织站点的默认发布源是存储库默认分支的根目录。项目站点的默认发布源是 gh-pages分支的根目录。

如果要将站点的源文件保存在其他位置，可以更改站点的发布源。 您可以从存储库中的任何分支发布站点，可以从该分支上存储库的根目录/或该分支上的/docs文件夹发布。 更多信息请参阅"Configuring a publishing source for your GitHub Pages site"(此链接网页未翻译)。

如果您选择任何分支的 /docs 文件夹作为您的发布源，GitHub Pages 将从 /docs 文件夹中读取所有内容以发布您的站点，包括 CNAME 文件。例如，当您通过 GitHub Pages 设置编辑自定义域时，自定义域将写入 /docs/CNAME。有关 CNAME 文件的更多信息，请参阅"Managing a custom domain for your GitHub Pages site"(此链接网页未翻译)。

 

 

Static site generators

GitHub Pages 发布您推送到存储库的任何静态文件。您可以创建自己的静态文件或使用静态站点生成器为您构建站点。您还可以在本地或另一台服务器上自定义您自己的构建过程。我们推荐 Jekyll，这是一个静态站点生成器，内置对 GitHub Pages 的支持和简化的构建过程。更多信息请参阅 "About GitHub Pages and Jekyll"(此链接网页未翻译)。

默认情况下，GitHub Pages 将使用 Jekyll 来构建您的站点。 如果您想使用 Jekyll 以外的静态站点生成器，请通过在发布源的根目录中创建一个名为 .nojekyll 的空文件来禁用 Jekyll 构建过程，然后按照静态站点生成器的说明在本地构建您的站点。

GitHub Pages 不支持服务器端语言，例如 PHP、Ruby 或 Python。

 

 

Guidelines for using GitHub Pages

  • 2016年6月15日之后创建并使用 github.io 域的 GitHub Pages 站点通过 HTTPS 提供服务。如果您在 2016年6月15日之前创建了您的网站，您可以为您网站的流量启用 HTTPS 支持。更多信息请参阅 "Securing your GitHub Pages with HTTPS"(此链接网页未翻译)。
  • GitHub Pages 站点不应用于发送密码或信用卡号等敏感交易。
  • 您对 GitHub Pages 的使用受 GitHub 服务条款的约束，包括禁止转售。
使用限制

GitHub Pages 站点受以下使用限制的约束：

  • GitHub Pages 源存储库的建议限制为 1GB。 更多信息请参阅 "What is my disk quota? "(此链接网页未翻译)
  • 已发布的 GitHub Pages 站点不得超过 1 GB。
  • GitHub Pages 站点的软 带宽限制为每月 100GB。
  • GitHub Pages 站点的软 限制为每小时 10 次构建。

如果您的站点超过这些使用配额，我们可能无法为您的站点提供服务，或者您可能会收到来自 GitHub Support(此链接网页未翻译) 或 GitHub Premium Support(此链接网页未翻译) 的电子邮件，建议减少您的站点对我们服务器影响的策略，包括放置第三方内容分发网络 (CDN)，利用其他 GitHub 功能（例如发布），或转移到可能更适合您需求的其他托管服务。

禁止用途

GitHub Pages 不打算或不允许用作免费的网络托管服务来运行您的在线业务、电子商务网站或任何其他主要旨在促进商业交易或提供商业软件即服务 (SaaS) 的网站。

此外，GitHub 不允许将 GitHub Pages 用于某些目的或活动。有关禁止使用的列表，请参阅 "GitHub's Additional Product Terms for GitHub Pages"(此链接网页未翻译)。

 

 

MIME types on GitHub Pages

MIME 类型是服务器发送到浏览器的标头，提供有关浏览器请求的文件的性质和格式的信息。GitHub Pages 支持跨越数千个文件扩展名的 750 多种 MIME 类型。支持的 MIME 类型列表是从 mime-db(此链接网页未翻译) 项目生成的。

虽然您不能在每个文件或每个存储库的基础上指定自定义 MIME 类型，但您可以添加或修改 MIME 类型以在 GitHub 页面上使用。有关更多信息，请参阅 the mime-db contributing guidelines(此链接网页未翻译)。
